Q4 Python is an interpreted language – explain in detail. 3M

An interpreted language executes code line by line at runtime, without a separate compilation step into machine code.

How Python Interpretation Works:

  Source Code (.py)
        ↓
  Python Interpreter (CPython)
        ↓
  Bytecode (.pyc) — stored in __pycache__
        ↓
  Python Virtual Machine (PVM)
        ↓
  Output / Result
  • The interpreter reads each line, converts it to intermediate bytecode, then the PVM executes it.
  • Errors are detected at runtime, line by line.
  • Easier debugging – you see errors as they occur.
  • Portability – same .py file runs on any OS with Python installed.
  • Trade-off: slower execution compared to compiled languages like C.
Note: CPython is the default interpreter. Others include PyPy (JIT-compiled for speed), Jython (runs on JVM), and IronPython (.NET).

Q12 What is dictionary in Python? Explain with an example. 6M

A dictionary is an unordered (Python 3.7+ maintains insertion order), mutable collection of key-value pairs. Keys must be unique and immutable; values can be any type.

# Creating a dictionary
student = {
    "name"  : "Alice",
    "age"   : 20,
    "marks" : [85, 90, 78]
}

# Accessing values
print(student["name"])       # Alice
print(student.get("age"))    # 20

# Adding / Updating
student["city"] = "Ahmedabad"
student["age"]  = 21

# Deleting
del student["city"]

# Useful methods
print(student.keys())    # dict_keys(['name', 'age', 'marks'])
print(student.values())  # dict_values(['Alice', 21, [85,90,78]])
print(student.items())   # key-value pairs as tuples

# Iterating
for key, val in student.items():
    print(f"{key}: {val}")
Important: Dictionary keys must be immutable (strings, numbers, tuples). Lists cannot be keys.

Q16 Explain List Comprehension. 4M

List Comprehension provides a concise way to create lists from iterables, replacing traditional for loops.

Syntax:

[expression  for  item  in  iterable  if  condition]
# Traditional way
squares = []
for x in range(1, 6):
    squares.append(x**2)

# List Comprehension (one line!)
squares = [x**2 for x in range(1, 6)]
print(squares)   # [1, 4, 9, 16, 25]

# With condition (filter even numbers)
evens = [x for x in range(10) if x % 2 == 0]
print(evens)     # [0, 2, 4, 6, 8]

# Nested list comprehension (matrix flattening)
matrix = [[1,2,3],[4,5,6],[7,8,9]]
flat = [num for row in matrix for num in row]
print(flat)      # [1,2,3,4,5,6,7,8,9]
Tip: List comprehensions are faster than equivalent for loops because they are optimised at the C level in CPython.

Q33 Explain the while loop with an example. 4M

A while loop repeatedly executes a block of code as long as its condition is True.

# Syntax:
# while condition:
#     body

# Example 1: Print 1 to 5
i = 1
while i <= 5:
    print(i, end=" ")
    i += 1
# Output: 1 2 3 4 5

# Example 2: Sum until user enters 0
total = 0
num = int(input("Enter number (0 to stop): "))
while num != 0:
    total += num
    num = int(input("Enter number (0 to stop): "))
print("Sum:", total)
Warning: Always ensure the loop condition eventually becomes False, or include a break statement. An infinite loop freezes the program.

Q51 Explore in detail about range function with suitable examples. 6M

range() generates a sequence of numbers. It is most commonly used with for loops.

Syntax: range(start, stop, step)

# range(stop) – 0 to stop-1
for i in range(5):
    print(i, end=" ")       # 0 1 2 3 4

# range(start, stop)
for i in range(2, 7):
    print(i, end=" ")       # 2 3 4 5 6

# range(start, stop, step)
for i in range(0, 20, 5):
    print(i, end=" ")       # 0 5 10 15

# Negative step (countdown)
for i in range(10, 0, -2):
    print(i, end=" ")       # 10 8 6 4 2

# Convert to list
print(list(range(1, 6)))    # [1, 2, 3, 4, 5]

# Check membership
print(4 in range(1, 6))     # True
Tip: range() does NOT store all numbers in memory – it generates them on demand, making it memory-efficient for large ranges.

Q77 Explain Python recursive function with an example. 6M

A recursive function calls itself to solve smaller sub-problems of the same type.

Two essential parts:

  • Base Case: stops the recursion (prevents infinite loop)
  • Recursive Case: function calls itself with a smaller problem
# Factorial using recursion
def factorial(n):
    if n == 0 or n == 1:   # Base case
        return 1
    return n * factorial(n - 1)  # Recursive case

print(factorial(5))   # 120
# factorial(5) = 5 * factorial(4)
#              = 5 * 4 * factorial(3)
#              = 5 * 4 * 3 * factorial(2)
#              = 5 * 4 * 3 * 2 * factorial(1)
#              = 5 * 4 * 3 * 2 * 1 = 120

# Fibonacci using recursion
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

for i in range(8):
    print(fibonacci(i), end=" ")   # 0 1 1 2 3 5 8 13
Warning: Deep recursion can cause RecursionError (default limit ~1000 calls). Python is not optimized for tail recursion. Use iteration for performance-critical code.

import numpy as np

view = original[1:4]
view[0] = 99
