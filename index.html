Q5
What is the significance of using placeholder attributes in form inputs with example?
3 Marks

The placeholder attribute provides a short hint that describes the expected value of an input field. It appears inside the field when it is empty and disappears when the user starts typing.

Significance:

  • Provides contextual hints about expected input format
  • Improves user experience and form usability
  • Reduces need for extra instruction text near inputs
  • Works with text, email, password, search, tel, url, textarea
<form>
  <input type="text"     placeholder="Enter your full name">
  <input type="email"    placeholder="example@gmail.com">
  <input type="tel"      placeholder="+91 9876543210">
  <input type="password" placeholder="Minimum 8 characters">
  <textarea              placeholder="Write your message here..."></textarea>
</form>
📌 Note: Placeholder should NOT replace <label> for accessibility. Always use both for screen reader support.

Q9
What is Grail layout in HTML?
8 Marks

The Holy Grail Layout is a classic page layout consisting of a header, footer, a main content area, and two sidebars. Historically hard to achieve with CSS floats, it's now easy with Flexbox or Grid.

💡 Called "Holy Grail" because it was the ideal layout that was elusive to achieve — like the mythical Holy Grail!

Structure Diagram:

+---------------------------------------+
|             HEADER                    |
+----------+---------------+-----------+
|          |               |           |
|  LEFT    |    MAIN       |   RIGHT   |
| SIDEBAR  |   CONTENT     |  SIDEBAR  |
|          |               |           |
+----------+---------------+-----------+
|             FOOTER                    |
+---------------------------------------+

Implementation using Flexbox:

<!DOCTYPE html>
<html>
<head>
<style>
  * { margin: 0; padding: 0; box-sizing: border-box; }
  body {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
  }
  header, footer {
    background: #333; color: white;
    padding: 20px; text-align: center;
  }
  .middle { display: flex; flex: 1; }
  nav    { background: #555; color: white; width: 180px; padding: 20px; }
  main   { flex: 1; padding: 20px; background: #f0f0f0; }
  aside  { background: #777; color: white; width: 180px; padding: 20px; }
</style>
</head>
<body>
  <header>Site Header</header>
  <div class="middle">
    <nav>Left Sidebar</nav>
    <main>Main Content</main>
    <aside>Right Sidebar</aside>
  </div>
  <footer>Site Footer</footer>
</body>
</html>

Implementation using CSS Grid (preferred):

<style>
  body {
    display: grid;
    grid-template-areas:
      "header header header"
      "nav    main   aside"
      "footer footer footer";
    grid-template-columns: 200px 1fr 200px;
    grid-template-rows: auto 1fr auto;
    min-height: 100vh;
  }
  header { grid-area: header; background: #333; color: white; padding: 20px; }
  nav    { grid-area: nav;    background: #555; color: white; padding: 20px; }
  main   { grid-area: main;   padding: 20px; }
  aside  { grid-area: aside;  background: #777; color: white; padding: 20px; }
  footer { grid-area: footer; background: #333; color: white; padding: 20px; }
</style>
📌 Grid method is more readable and powerful — preferred for Holy Grail layouts in modern CSS.

Q16
Define: Canvas, Figma, Hierarchy, Components, Prototyping
5 Marks
TermDefinition
1. Canvas In HTML5, <canvas> is an element for drawing 2D graphics via JavaScript API. In design tools (Figma/Adobe XD), the canvas is the infinite workspace where UI designs are created and arranged.
2. Figma A cloud-based, collaborative UI/UX design tool used for creating interfaces, prototypes, and design systems. Works in browser — no installation needed. Supports real-time multi-user collaboration, vector editing, and component libraries.
3. Hierarchy Visual hierarchy is the arrangement of design elements to indicate their order of importance. Achieved through: size, color contrast, typography weight, spacing, and position. Guides the user's eye in a logical sequence through content.
4. Components Reusable design or code elements created once and used multiple times. In Figma: a component (e.g., Button, Card) is created as a master and reused as instances. Editing the master component updates all instances automatically. Promotes design consistency.
5. Prototyping Creating an interactive simulation/mockup of a design to test user flows before development. In Figma: frames are connected with interaction arrows to simulate navigation. Types: Low-fidelity (wireframes) and High-fidelity (pixel-perfect). Enables early user testing and stakeholder feedback.

Flexbox (Flexible Box Layout) is a CSS layout module that provides a one-dimensional layout system for arranging items in rows or columns.

Flex Container (Parent)
+-----------------------------------------------+
|  Main Axis →  (row) or ↓ (column)            |
|  +--------+  +--------+  +--------+           |
|  | Item 1 |  | Item 2 |  | Item 3 |           |
|  +--------+  +--------+  +--------+           |
|                               ↕ Cross Axis    |
+-----------------------------------------------+

Container Properties (applied to parent):

.container {
  display: flex;                        /* Activate flexbox */

  flex-direction: row;                  /* row | row-reverse | column | column-reverse */
  flex-wrap: wrap;                      /* nowrap | wrap | wrap-reverse */
  flex-flow: row wrap;                  /* shorthand for direction + wrap */

  justify-content: center;             /* Main axis alignment */
  /* flex-start | flex-end | center | space-between | space-around | space-evenly */

  align-items: center;                  /* Cross axis alignment */
  /* flex-start | flex-end | center | stretch | baseline */

  align-content: space-between;        /* Multiple lines alignment */
  gap: 20px;                           /* Space between items */
  gap: 10px 20px;                      /* row-gap column-gap */
}

Item Properties (applied to children):

.item {
  flex-grow: 1;    /* How much item grows to fill space (0 = don't grow) */
  flex-shrink: 1;  /* How much item shrinks (0 = don't shrink) */
  flex-basis: auto;/* Initial size before growing/shrinking */
  flex: 1;         /* Shorthand: grow=1 shrink=1 basis=0 */

  align-self: flex-end;  /* Override parent align-items for this item */
  order: 2;              /* Display order (default: 0) */

What is grid layout? Explain grid layout in detail with example.
8 Marks

CSS Grid Layout is a two-dimensional layout system that allows control over both rows and columns simultaneously. It's the most powerful CSS layout tool.

Grid Container
+--------+--------+--------+
| col 1  | col 2  | col 3  |  ← Row 1
+--------+--------+--------+
| col 1  | col 2  | col 3  |  ← Row 2
+--------+--------+--------+
    ↑           ↑
  Grid Line   Grid Area

Container Properties:

.container {
  display: grid;

  /* Define columns */
  grid-template-columns: 200px 1fr 200px;          /* 3 columns */
  grid-template-columns: repeat(3, 1fr);           /* Equal 3 cols */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); /* Responsive */

  /* Define rows */
  grid-template-rows: 80px 1fr 60px;

  /* Named areas */
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";

  /* Gaps */
  gap: 20px;               /* row and column gap */
  row-gap: 15px;
  column-gap: 25px;

  /* Alignment */
  justify-items: center;   /* Horizontal alignment of items */
  align-items: center;     /* Vertical alignment of items */
}

Item Properties:

.item {
  grid-column: 1 / 3;    /* Start at col line 1, end at 3 */
  grid-column: span 2;   /* Span 2 columns */
  grid-row: 1 / 3;       /* Span 2 rows */
  grid-area: header;     /* Assign to named area */
  justify-self: end;     /* Override horizontal alignment */
  align-self: start;     /* Override vertical alignment */
}
