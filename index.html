Q9
What is Grail layout in HTML?
8 Marks

The Holy Grail Layout is a classic page layout consisting of a header, footer, a main content area, and two sidebars. Historically hard to achieve with CSS floats, it's now easy with Flexbox or Grid.

💡 Called "Holy Grail" because it was the ideal layout that was elusive to achieve — like the mythical Holy Grail!

Structure Diagram:

+---------------------------------------+
|              HEADER                   |
+----------+---------------+-----------+
|          |               |           |
|  LEFT    |    MAIN       |   RIGHT   |
| SIDEBAR  |   CONTENT     |  SIDEBAR  |
|          |               |           |
+----------+---------------+-----------+
|              FOOTER                   |
+---------------------------------------+

Implementation using Flexbox:

<!DOCTYPE html>
<html>
<head>
<style>
  * { margin: 0; padding: 0; box-sizing: border-box; }
  body {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
  }
  header, footer {
    background: #333; color: white;
    padding: 20px; text-align: center;
  }
  .middle { display: flex; flex: 1; }
  nav    { background: #555; color: white; width: 180px; padding: 20px; }
  main   { flex: 1; padding: 20px; background: #f0f0f0; }
  aside  { background: #777; color: white; width: 180px; padding: 20px; }
</style>
</head>
<body>
  <header>Site Header</header>
  <div class="middle">
    <nav>Left Sidebar</nav>
    <main>Main Content</main>
    <aside>Right Sidebar</aside>
  </div>
  <footer>Site Footer</footer>
</body>
</html>

Implementation using CSS Grid (preferred):

<style>

  main   { grid-area: main;   padding: 20px; }

  • Reduces code repetition — one stylesheet for entire site

Q3
Explain CSS box model (border, margin, padding) with neat diagram.
8 Marks

The CSS Box Model describes how every HTML element is rendered as a rectangular box consisting of four layers.

+----------------------------------------------+
|                  MARGIN                      |  ← Transparent space outside border
|   +--------------------------------------+   |
|   |              BORDER                  |   |  ← Visible boundary
|   |   +------------------------------+   |   |
|   |   |           PADDING            |   |   |  ← Space between content & border
|   |   |   +----------------------+   |   |   |
|   |   |   |       CONTENT        |   |   |   |  ← Text, images, etc.
|   |   |   |  (width × height)    |   |   |   |
|   |   |   +----------------------+   |   |   |
|   |   +------------------------------+   |   |
|   +--------------------------------------+   |
+----------------------------------------------+
LayerPropertyDescription
Contentwidth, heightActual element content area
Paddingpadding-top/right/bottom/leftSpace between content and border (inherits background)
Borderborder-width/style/colorSurrounds padding — can be styled
Marginmargin-top/right/bottom/leftSpace outside border — always transparent
/* Full Box Model Example */
div {
  width: 200px;        /* Content width */
  height: 100px;       /* Content height */

  padding: 20px;       /* All 4 sides */
  padding: 10px 20px;  /* top/bottom | left/right */

  border: 2px solid red;
  border-radius: 8px;  /* Rounded corners */

  margin: 30px;         /* All 4 sides */
  margin: 10px auto;    /* top/bottom | center horizontally */
}

/* Total element size = content + padding + border */
/* width: 200 + 20+20 (pad) + 2+2 (border) = 244px */

box-sizing property:

/* Default - padding/border ADD to specified width */
box-sizing: content-box;   /* width = content only */

/* Width INCLUDES padding and border (recommended) */
box-sizing: border-box;    /* Easier to manage layouts */

/* Best practice - apply to all elements */
* { box-sizing: border-box; }
📌 Use browser DevTools (F12) to visually inspect the box model of any element.

Q10
Explain Flex layout in detail with example.
8 Marks

Flexbox (Flexible Box Layout) is a CSS layout module that provides a one-dimensional layout system for arranging items in rows or columns.

Flex Container (Parent)
+-----------------------------------------------+
|  Main Axis →  (row) or ↓ (column)             |
|  +--------+  +--------+  +--------+           |
|  | Item 1 |  | Item 2 |  | Item 3 |           |
|  +--------+  +--------+  +--------+           |
|                                ↕ Cross Axis   |
+-----------------------------------------------+

Container Properties (applied to parent):

.container {
  display: flex;                        /* Activate flexbox */

  flex-direction: row;                  /* row | row-reverse | column | column-reverse */
  flex-wrap: wrap;                      /* nowrap | wrap | wrap-reverse */
  flex-flow: row wrap;                  /* shorthand for direction + wrap */

  justify-content: center;             /* Main axis alignment */
  /* flex-start | flex-end | center | space-between | space-around | space-evenly */

  align-items: center;                  /* Cross axis alignment */
  /* flex-start | flex-end | center | stretch | baseline */

  align-content: space-between;        /* Multiple lines alignment */
  gap: 20px;                           /* Space between items */
  gap: 10px 20px;                      /* row-gap column-gap */
}

Item Properties (applied to children):

.item {
  flex-grow: 1;    /* How much item grows to fill space (0 = don't grow) */
  flex-shrink: 1;  /* How much item shrinks (0 = don't shrink) */
  flex-basis: auto;/* Initial size before growing/shrinking */
  flex: 1;         /* Shorthand: grow=1 shrink=1 basis=0 */

  align-self: flex-end;  /* Override parent align-items for this item */
  order: 2;              /* Display order (default: 0) */
}

Practical Example — Navigation Bar:

What is grid layout? Explain grid layout in detail with example.
8 Marks

CSS Grid Layout is a two-dimensional layout system that allows control over both rows and columns simultaneously. It's the most powerful CSS layout tool.

Grid Container
+--------+--------+--------+
| col 1  | col 2  | col 3  |  ← Row 1
+--------+--------+--------+
| col 1  | col 2  | col 3  |  ← Row 2
+--------+--------+--------+
    ↑           ↑
  Grid Line   Grid Area

Container Properties:

.container {
  display: grid;

  /* Define columns */
  grid-template-columns: 200px 1fr 200px;          /* 3 columns */
  grid-template-columns: repeat(3, 1fr);           /* Equal 3 cols */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); /* Responsive */

  /* Define rows */
  grid-template-rows: 80px 1fr 60px;

  /* Named areas */
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";

  /* Gaps */
  gap: 20px;               /* row and column gap */
  row-gap: 15px;
  column-gap: 25px;

  /* Alignment */
  justify-items: center;   /* Horizontal alignment of items */
  align-items: center;     /* Vertical alignment of items */
}

Item Properties:

.item {
  grid-column: 1 / 3;    /* Start at col line 1, end at 3 */
  grid-column: span 2;   /* Span 2 columns */
  grid-row: 1 / 3;       /* Span 2 rows */
  grid-area: header;     /* Assign to named area */
  justify-self: end;     /* Override horizontal alignment */
  align-self: start;     /* Override vertical alignment */
}

Complete Example — Page Layout:

<style>

    display: grid;
    grid-template-areas:
      "header header"
      "sidebar content"

  .sidebar { grid-area: sidebar; background: #555; color: white; }

What would be the result of 3 + 2 + "7"?

Result: "57"

JavaScript evaluates expressions left-to-right:

3 + 2 + "7"

Step 1: 3 + 2 = 5         (both numbers → arithmetic addition)
Step 2: 5 + "7" = "57"    (number + string → string concatenation)

Final Result: "57"  (a STRING, not number 12)

Why does this happen? — Type Coercion

When the + operator encounters a string, it converts the other operand to a string and concatenates. Left-to-right evaluation means 3+2 happens first (=5), then 5+"7" becomes "57".

console.log(3 + 2 + 7);      // 12   (all numbers)

What is Traversing?

Traversing in jQuery means moving through (navigating) the DOM tree to find, filter, or select elements based on their relationship to other elements.

DirectionMethods
Go Up (ancestors).parent(), .parents(), .parentsUntil(), .closest()
Go Down (descendants).children(), .find()
Go Sideways (siblings).siblings(), .next(), .nextAll(), .nextUntil(), .prev(), .prevAll()
Filter.filter(), .not(), .has(), .is(), .eq(), .first(), .last()
<div id="container">
  <h2>Title</h2>

    <li class="item">Item 1</li>

  </ul>
</div>

// Traversal examples
var activeItem = $(".active");

activeItem.parent();      // → <ul id="list">
activeItem.parents();     // → ul, div#container, body, html
activeItem.closest("div"); // → div#container

activeItem.siblings();    // → Item 1, Item 3
activeItem.next();        // → Item 3 (next sibling)
activeItem.prev();        // → Item 1 (previous sibling)

$("#container").find(".active"); // → Item 2

$(".item").filter(":odd");     // Items at odd index
$(".item").not(".active");     // Items without 'active' class
$(".item").eq(0);              // First item (index 0)
