Q1 Explain features of JAVA. 3 marks ▼

Key Features of Java

  • Simple: Java has a clean, easy-to-learn syntax derived from C/C++ with complex features (like pointers, operator overloading) removed.
  • Object-Oriented: Everything in Java is an object. Supports encapsulation, inheritance, polymorphism, and abstraction.
  • Platform Independent (WORA): Write Once, Run Anywhere. Java bytecode runs on any machine with a JVM.
  • Secure: No pointer arithmetic; bytecode is verified before execution; has a Security Manager.
  • Robust: Strong type checking, exception handling, garbage collection — reduces runtime errors.
  • Multithreaded: Built-in support for concurrent programming via Thread class and Runnable interface.
  • Architecture Neutral: Compiled to architecture-neutral bytecode (.class files) independent of CPU.
  • High Performance: Just-In-Time (JIT) compiler translates bytecode to native machine code at runtime.
  • Distributed: Rich set of networking libraries (java.net) for building distributed applications.
  • Dynamic: Supports dynamic loading of classes and runtime binding.
WORA = Write Once, Run Anywhere — Java's core promise enabled by the JVM.

JDK — Java Development Kit

The complete development package. Contains: compiler (javac), debugger (jdb), archiver (jar), documentation generator (javadoc), and the JRE. Used by developers to write and compile Java programs.

JRE — Java Runtime Environment

Provides the runtime environment to execute Java programs. Contains: JVM + Java Class Libraries (java.lang, java.util, java.io, etc.) + supporting files. End-users who only run Java apps need the JRE, not full JDK.

JVM — Java Virtual Machine

An abstract computing machine that executes Java bytecode (.class files). Key components:

  • Class Loader: Loads .class files into memory
  • Bytecode Verifier: Checks code for security violations
  • Interpreter: Executes bytecode instructions
  • JIT Compiler: Compiles hot bytecode to native machine code
  • Garbage Collector: Automatically reclaims unused memory
JVM is platform-specific (different JVMs for Windows, Linux, Mac), but the bytecode it runs is platform-independent. This is how WORA works!

Q8 Explain type casting with an example. 4 marks ▼
Type Casting is the process of converting a value from one data type to another.

Types of Casting

1. Widening (Implicit) Casting

Converting a smaller type to a larger type — done automatically by Java. No data loss.

byte → short → int → long → float → double

int i = 100;
double d = i;    // Implicit: int → double
System.out.println(d); // 100.0

2. Narrowing (Explicit) Casting

Converting a larger type to a smaller type — must be done manually. May cause data loss.

double → float → long → int → short → byte

double d = 9.78;
int i = (int) d;   // Explicit: double → int
System.out.println(i); // 9 (decimal part lost!)

3. Object Casting (Upcasting / Downcasting)

Animal a = new Dog();        // Upcasting (implicit)
Dog d = (Dog) a;             // Downcasting (explicit)
Narrowing cast can cause data loss or overflow. Always check the range before narrowing.

The switch statement selects one of many code blocks to execute based on the value of an expression. It is a cleaner alternative to multiple if-else-if chains.

Syntax

switch (expression) {
    case value1:
        // code block
        break;
    case value2:
        // code block
        break;
    default:
        // default block (optional)
}

    case 1: System.out.println("Monday");    break;
    case 2: System.out.println("Tuesday");   break;
    case 3: System.out.println("Wednesday"); break;
    case 4: System.out.println("Thursday");  break;

Switch works with: int, char, String (Java 7+), enum. Does NOT work with float, double, long.

Q17 Write a program to do matrix multiplication using multidimensional array. 8 marks ▼

Matrix Multiplication Rule

C[i][j] = Σ A[i][k] * B[k][j]  for k = 0 to n-1

A(m×n) × B(n×p) = C(m×p)
public class MatrixMultiply {
    public static void main(String[] args) {
        int[][] A = {
            {1, 2, 3},
            {4, 5, 6}
        };  // 2×3 matrix

        int[][] B = {
            {7,  8},
            {9,  10},
            {11, 12}
        };  // 3×2 matrix

        int rows = A.length;
        int cols = B[0].length;
        int inner = B.length;

        int[][] C = new int[rows][cols];  // 2×2 result

        // Multiply
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                for (int k = 0; k < inner; k++) {
                    C[i][j] += A[i][k] * B[k][j];
                }
            }
        }

        // Print result
        System.out.println("Result Matrix C:");
        for (int[] row : C) {
            for (int val : row)
                System.out.printf("%5d", val);
            System.out.println();
        }
    }
}

Q19 What is a Constructor? Explain types. Program to find Area of Circle using Constructor. 8 marks ▼
A constructor is a special method that initializes an object when it is created. It has the same name as the class and no return type.

Types of Constructors

1. Default Constructor

No parameters. Java automatically provides one if you define no constructors. Assigns default values (0, null, false).

2. Parameterized Constructor

Takes arguments to initialize object with specific values.

3. Copy Constructor

Takes an object of the same class as parameter and creates a copy.

public class Circle {
    double radius;

    // Default constructor
    Circle() {
        radius = 1.0;
    }

    // Parameterized constructor
    Circle(double r) {
        radius = r;
    }

    // Copy constructor
    Circle(Circle c) {
        radius = c.radius;
    }

    double area() {
        return Math.PI * radius * radius;
    }

    public static void main(String[] args) {
        Circle c1 = new Circle();        // radius = 1.0
        Circle c2 = new Circle(5.0);    // radius = 5.0
        Circle c3 = new Circle(c2);    // copy of c2

        System.out.printf("c1 area = %.4f%n", c1.area()); // 3.1416
        System.out.printf("c2 area = %.4f%n", c2.area()); // 78.5398
        System.out.printf("c3 area = %.4f%n", c3.area()); // 78.5398
    }
}

this Keyword

Refers to the current object. Used to: distinguish instance variables from parameters, call other constructors (this()), return the current object.

class Box {
    int width;

static Keyword

Members declared static belong to the class, not individual objects. Shared among all instances. Can be accessed without creating an object.

class Counter {
    static int count = 0;  // shared
    Counter() { count++; }
    static int getCount() { return count; }
}
// Counter.getCount() — no object needed

Garbage Collection & finalize()

Java's Garbage Collector (GC) automatically frees memory of objects no longer referenced. The finalize() method in the Object class is called by the GC before collecting the object — used for cleanup (closing files, connections). You can request GC with System.gc() but it's not guaranteed.

class Resource {
    protected void finalize() {
        System.out.println("Cleaning up resource...");
    }
}
Resource r = new Resource();
r = null;       // eligible for GC
System.gc();    // suggest GC run
finalize() is deprecated in Java 9+. Use try-with-resources or explicit close() instead.

Q22 What is a constructor? Rules for creating constructors in Java. 4 marks ▼
A constructor is a special block of code that initializes a newly created object. It is called automatically by the JVM when an object is instantiated using new.

Rules for Creating Constructors

  • Constructor name must be identical to the class name
  • Constructor has no return type — not even void
  • Can have any access modifier: public, protected, private, default
  • Can be overloaded (multiple constructors with different parameters)
  • Cannot be static, final, abstract, or synchronized
  • If you define no constructor, Java provides a default (no-arg) constructor
  • If you define any constructor, Java does NOT provide the default one automatically
  • A constructor can call another constructor using this() — must be first statement
  • A child constructor can call parent constructor using super() — must be first statement

    // ❌ Invalid — has return type (this is a METHOD not a constructor)

}

        try (FileInputStream  in  = new FileInputStream("source.txt");
             FileOutputStream out = new FileOutputStream("dest.txt")) {
            byte[] buffer = new byte[1024];
            int bytesRead;
            while ((bytesRead = in.read(buffer)) != -1) {
                out.write(buffer, 0, bytesRead);
            }

        }

import java.util.*;

public class StudentGrades {
    private HashMap<String, Double> grades = new HashMap<>();

    // Add new grade
    public void addGrade(String student, double grade) {
        if (grades.containsKey(student)) {
            System.out.println(student + " already exists. Use updateGrade().");
        } else {
            grades.put(student, grade);
            System.out.println("Added: " + student + " → " + grade);
        }
    }

    // Update existing grade
    public void updateGrade(String student, double newGrade) {
        if (grades.containsKey(student)) {
            grades.put(student, newGrade);
            System.out.println("Updated: " + student + " → " + newGrade);
        } else {
            System.out.println("Student not found: " + student);
        }
    }

    // Calculate average grade
    public double calculateAverage() {
        if (grades.isEmpty()) return 0.0;
        double sum = 0;
        for (double g : grades.values()) sum += g;
        return sum / grades.size();
    }

    // Display all grades
    public void displayAll() {
        System.out.println("\n--- Student Grades ---");
        for (Map.Entry<String, Double> entry : grades.entrySet()) {
            System.out.printf("%-15s: %.2f%n",
                entry.getKey(), entry.getValue());
        }
        System.out.printf("Average: %.2f%n", calculateAverage());
    }

    public static void main(String[] args) {
        StudentGrades sg = new StudentGrades();

        sg.addGrade("Alice", 88.5);
        sg.addGrade("Bob",   72.0);
        sg.addGrade("Carol", 95.0);

        sg.updateGrade("Bob", 78.5);

        sg.displayAll();
    }
}

Q70 What does the term object oriented mean? Explain the four aspects of an OO approach. 4 marks ▼
Object-Oriented (OO) is a programming paradigm based on organizing software as a collection of interacting objects, each encapsulating data and behavior. It models real-world entities.

Four Key OO Concepts

1. Encapsulation

Bundling data (fields) and methods that operate on that data within a single unit (class). Hides internal details. Implemented via access modifiers.

2. Inheritance

A class (subclass) acquires properties and behavior from another class (superclass). Promotes code reuse. Establishes IS-A relationship.

3. Polymorphism

Ability of an object to take multiple forms. The same method behaves differently based on the object calling it. Supports flexible and extensible code.

4. Abstraction

Hiding complex implementation details and showing only essential features. Achieved through abstract classes and interfaces.

// All four in one example
abstract class Shape {           // Abstraction
    private String color;          // Encapsulation
    abstract double area();        // Must implement
    void setColor(String c) { color = c; }
}

class Circle extends Shape {     // Inheritance
    double r;
    double area() { return Math.PI*r*r; } // Polymorphism
}
Q71 Define and discuss OO characteristics: Identity, Classification, Inheritance, Polymorphism. 4 marks ▼

1. Identity

Every object has a unique identity that distinguishes it from other objects, even if they have the same attribute values. In Java, identity is provided by the object's memory reference (address). The == operator tests identity.

Dog d1 = new Dog(); Dog d2 = new Dog();
d1 == d2  // false — different identities, same class

2. Classification

Objects with the same structure (attributes) and behavior (methods) are grouped into a class. A class defines a type; objects are instances of that type. Classification helps manage complexity through abstraction levels.

3. Inheritance

A mechanism where a new class (subclass) is derived from an existing class (superclass). The subclass inherits attributes and methods and can add or override them. Enables code reuse and the IS-A relationship.

4. Polymorphism

An operation can behave differently when applied to different types of objects. Allows writing generic code that works with objects of multiple types through a common interface.

In UML Activity Diagrams, Fork splits a single flow into multiple concurrent flows. Join synchronizes multiple concurrent flows back into a single flow. Both are represented as thick horizontal or vertical bars.

Fork

A fork node has one incoming edge and multiple outgoing edges. All outgoing activities start concurrently (parallel execution).

Join

A join node has multiple incoming edges and one outgoing edge. The outgoing activity starts only when ALL incoming activities complete.
